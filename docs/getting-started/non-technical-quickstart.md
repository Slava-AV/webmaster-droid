# Non-Technical Quickstart

This quickstart is for editors who only need to update website content through the in-site admin overlay.

## Prerequisites

- You have an admin account for the overlay.
- The site already has Webmaster Droid integrated by a developer.
- Your team has an approval process for publishing.

## 1. Open the admin overlay

- Open your site in browser.
- Enable admin mode as configured by your team.
- Sign in through the overlay.

## 2. Request a change in plain language

Examples:

- `Update the About page intro to sound more friendly and concise.`
- `Generate a new banner image for spring promotions.`
- `Use the current hero image and make the look warmer.`
- `Fix this section text to be easier for first-time customers.`

## 3. Review draft updates

- Confirm the assistant summary.
- Check in-page visual result.
- Use History tab to review recent checkpoints.

## 4. Publish explicitly

- Publish only after review.
- Remember: draft changes are not live until publish.

## 5. If something looks wrong

- Restore a previous checkpoint from History.
- Re-run request with clearer instructions.
- Escalate to developer support for structural or schema-level changes.
